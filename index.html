Step 1
Set up your team repository
One person on your team does this. Everyone else joins. Takes about 5 minutes.
What's a repository? Think of it as a shared Google Drive folder for your code. Everyone on your team can see it, edit it, and sync their changes.
One person creates the repository
Go to github.com, click the "+" in the top right corner, and select "New repository." Give it a name like "greencode-2026." Set it to Public and click "Create repository."
Add your teammates
Go to Settings → Collaborators → Add people. Type in your teammates' GitHub usernames and send them invites.
Everyone accepts the invite
Teammates check their email or GitHub notifications and accept the collaboration invite. Now everyone has access.
Clone the repo to your computer
In GitHub Desktop: File → Clone Repository → find yours → choose where to save it. Now the folder is on your laptop and ready to code in.
The golden rule: pull before you code, push after you code
Every time you sit down to work, hit "Fetch" in GitHub Desktop first to get your teammates' latest changes. When you're done writing something, hit "Commit" and "Push" to send your changes up. This avoids 99% of merge conflicts.
Pull → Write code → Commit → Push. Repeat.

Keep in mind
What judges actually look for
Don't spend 6 hours coding and 30 seconds presenting. Both matter.
Problem and relevance (20 pts) — Can you explain in one sentence what problem you're solving and who has it?
Technical execution (25 pts) — Does your project actually work during the demo? A simple working product beats an ambitious broken one.
Innovation and creativity (20 pts) — Did you make at least one interesting or unexpected decision? You don't need to reinvent the wheel, just make it yours.
User impact (15 pts) — Who benefits from this? Be specific. "Everyone" is not an answer. "High schoolers who get phishing emails" is.
Presentation (10 pts) — Judges should understand what you built within the first 30 seconds. Practice saying it out loud once before you go up.
